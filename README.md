# VOC 기반 Cartier 리스크 탐지 · 확산 모니터링 시스템 구축

## 프로젝트 배경

- 시크먼트(네이버 카페, 회원 79만, 월 MAU 100만↑) — 대한민국 최대 명품 커뮤니티
- 까르띠에: 한국 하이엔드 주얼리 1위, 시크먼트 내 언급 최다 브랜드
- 2025년 4차례 가격 인상 → 발표 직후 게시물 폭발적 반응 확인
- 이미 리스크 확산 중이지만 즉각 캐치 불가 → 브랜드 이미지 타격 심화
- **VOC 기반 리스크 트래킹 시스템 구축 필요**

## 프로젝트 개요

- 분석 기간: 2025년 1월 ~ 8월
- 데이터: 시크먼트 전체 11,969개 → 까르띠에 게시글 5,296개 → 리스크 게시글 1,860개
- 핵심 성과: BERT 이진분류로 5,296개 → 1,860개 (약 3배 축소), 9등급 매트릭스로 1,860개 → 404개 압축 → **전수 대비 선택적 대응으로 운영 효율 약 4배 향상**

## 주요 작업 내용

### 1. BERT 기반 브랜드 분류 고도화

- 팀 학습 데이터에 타 브랜드 게시글 55개 혼입 → 학습 오염 확인 후 제거
- Active Learning 적용: 모델이 불확실하게 판단하는 구간 집중 샘플링 (436 → 530 → 626개)
- Accuracy 0.95 / Macro F1 0.85 달성

### 2. 리스크 이진 분류

- 일반 BERT → 커뮤니티 텍스트 특화 KcBERT로 교체
- Threshold 최적화로 Recall 유지하면서 Precision 개선
- 게시글: Precision 0.573 → 0.659 (+8.6%p), Recall 0.784 유지
- 댓글: Precision 0.667 / Recall 0.833 달성
- F2-Score 차이 0.005 이내 — 리스크 탐지 능력 유지

### 3. 리스크 카테고리 분류

- 팀 결과: BERTopic 직접 적용 → 전체 75% 분석 제외
- 재설계: Rule-based 대분류(품질/운영/평판/정책) 먼저 적용해 범위 구조화 → 분류된 443개에 한해 BERTopic 세부 분류 시행 (노이즈 감소, 토픽 품질 확보)
- 대분류 결과: 운영(1,181개) / 평판(363개) / 품질(165개) / 정책(151개)
- BERTopic 세부 분류: 응대불만, 웨이팅/오픈런, 기스우려, 입고대기 등 8개 토픽

### 4. 리스크 지표 설계 및 9등급 분류

- Risk Score = (0.5 × 리스크확률) + (0.3 × 감성강도) + (0.2 × 키워드점수)
- Exposure Score = 리스크댓글비율 + 좋아요수 + 조회수
- Risk Score × Exposure Score → HH~LL 9등급 분류
- 1,860개 → 집중 모니터링 404개 (HH+HM+MH+MM)

### 5. 인사이트

- TF-IDF 단독 분석 시 볼륨 1위 응대불만을 최우선 리스크로 오판할 가능성 → BERTopic + Exposure Score 결합으로 언급량은 적지만 확산 시 치명적인 정책리스크(Exposure 0.91, 1위) 정확히 탐지
- Topic Evolution으로 가격 인상 전후 리스크 토픽 변화 분석
- Figma 기반 대시보드 레이아웃 설계 + Tableau 대시보드 2종 제작 (월·카테고리 선택 시 KPI·그래프 동적 연동)
